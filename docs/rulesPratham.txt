PHASE 2 — VISUAL DESIGN SYSTEM
Explicit Non-Goals:
- No excessive animations
- No parallax-heavy scrolling
- No autoplay media
- No visual noise that competes with content

Visual Mood
Dark, precise, and technically confident. The design should feel like a developer who builds serious systems — not flashy, but unmistakably sharp. Inspired by Linear, Raycast, and Vercel's engineering blog.

Color Strategy
Default: Dark mode
Rationale: Aligns with developer culture, reduces eye strain, conveys systems-level sophistication.

Color Tokens
Background Layers:

bg-primary: #09090b (near-black, main canvas)
bg-secondary: #111113 (cards, elevated surfaces)
bg-tertiary: #1c1c1f (hover states, subtle elevation)
Text:

text-primary: #f4f4f5 (high contrast, body text)
text-secondary: #a1a1aa (muted, descriptions, metadata)
text-tertiary: #71717a (de-emphasized, labels, captions)

Light Mode Policy:
Light mode is an optional secondary theme.
Dark mode is the canonical design and must be implemented first.
Light mode is not required for MVP.

Accent Colors:
Accent Usage Rules:
- Accent colors must be used sparingly.
- Never use more than one accent color on a single screen.
- accent-primary is reserved strictly for primary CTAs, links, and active states.
- accent-glow is used only for hover, focus, or subtle emphasis.
- accent-success is used only for availability indicators or success metrics.

Accent Rationale:
Indigo (#6366f1) was chosen because:
- It reads uniquely and distinctly "AI/ML/systems"
- It pairs well with the near-black background without looking corporate
- It stands out as a differentiator for a builder with ML + systems depth

accent-primary: #6366f1 (indigo — modern, AI-adjacent, trust-inspiring)
accent-glow: #818cf8 (lighter indigo for hover/focus states)
accent-success: #22c55e (green, for availability badges or success metrics)

Borders & Dividers:

border-subtle: #27272a (structural dividers)
border-focus: #6366f1 (interactive element focus rings)

Typography
Typography Metrics:
- Heading line-height: 1.1–1.2
- Body line-height: 1.6–1.7
- Large headings may use slight negative letter-spacing for visual balance.

Heading Font:
'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif

Rationale: Clean, modern, excellent readability.
Weights: 600 (semi-bold) for primary headings, 500 (medium) for subheadings

Body Font:
'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif

Weights: 400 (regular) for body, 300 (light) for large descriptive text

Monospace Font:
'JetBrains Mono', 'Fira Code', 'Monaco', monospace

Rationale: For tech stack tags, code snippets, technical metadata.
Weight: 400 (regular)

Spacing & Layout Philosophy
Grid System:
12-column responsive grid with generous whitespace

Spacing Scale (rem-based):

Micro: 0.25rem, 0.5rem (tight padding, icon gaps)
Small: 1rem, 1.5rem (component internal spacing)
Medium: 2rem, 3rem (section padding, card gaps)
Large: 4rem, 6rem (section dividers, vertical rhythm)
XL: 8rem, 10rem (hero section breathing room)

Layout Principles:

Generous vertical rhythm — Let sections breathe
Left-aligned hierarchy — Scannable F-pattern for recruiters
Maximum content width: 1200px centered
Mobile-first progression — Stack cleanly at small sizes, expand on desktop

Design Inspirations
Linear — Minimal navigation, focus on content, subtle gradients
Vercel — Dark mode execution, typography hierarchy, technical confidence
Raycast — Clean monospace aesthetics, precision UI
GitHub Profile Pages — Developer-centric layout, project showcase

Design Principles
Content is king — Design serves the work, not the other way around
Subtle motion — Micro-interactions only (hover states, smooth transitions), no distractions
Scannable hierarchy — Recruiters should find keywords in 30 seconds
Progressive disclosure — Show enough to intrigue, link to details
Performance as design — Fast load, no bloat, no unnecessary assets

Performance Constraint:
Avoid heavy raster images and unnecessary assets.
Prefer vector graphics, CSS-based effects, and lightweight SVGs.
Performance is treated as a design feature.

PHASE 4 — INTERACTIONS & ANIMATIONS (DESIGN LEVEL)
Explicit Motion Non-Goals:
- No page transition animations
- No cursor-following effects
- No scroll-jacking
- No background video or animated noise
- No continuous animations outside subtle availability indicators

Animation Philosophy
Guiding Principle: Motion should provide feedback, guidance, and delight—never decoration.
Motion Severity Levels:
- Level 0: No motion (when prefers-reduced-motion is enabled)
- Level 1: Micro-interactions only (hover, focus, feedback)
- Level 2: Entry animations (fade + slide only)
- Level 3: Expand/collapse interactions (projects, accordions)

This portfolio must not exceed Level 3 motion at any point.

Performance Constraints:

Total page weight budget: <500KB initial load
Animations must run at 60fps on mid-range devices
Use CSS transforms (translate, scale, opacity) over layout-affecting properties
Avoid animating width, height, top, left

Section-by-Section Motion Design

Scroll Trigger Rule:
Scroll-triggered animations run only once per page load.
Animations must not replay when scrolling back up.
